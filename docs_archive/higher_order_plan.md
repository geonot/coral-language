# Higher-Order Function & Placeholder Plan

## Goals
- Allow inline lambda expressions inside any expression position.
- Desugar placeholder syntax (e.g., `$ * 1.15`, `$1 + $2`) into explicit lambdas so downstream stages only see regular AST nodes.
- Provide runtime hooks so `list.map`, `list.reduce`, and user-defined higher-order APIs can call compiled Coral lambdas through a uniform ABI.

## Language Model
1. **Lambda literal**: `*fn(params)` followed by an indented block (same layout rules as functions). Single-expression blocks can stay inline, e.g. `*fn(value) value * 2`.
2. **Placeholder sugar**:
   - `$` expands to lambda with a single parameter `_0` when it appears inside an expression that needs a callable (parser transforms argument boundary to `Lambda` node with autogenerated parameter list).
   - `$1`, `$2`, ... expand to positional parameters `_0`, `_1`, ... for multi-argument contexts like `reduce`.
   - Placeholder scopes cannot escape: referencing `$` outside lowering context emits a diagnostic.

## AST Extensions
- `Expression::Lambda { params: Vec<Parameter>, body: Block, span: Span }`.
- `Expression::PlaceholderRef { index: u32, span: Span }` (parser uses this before desugaring; later pass rewrites to `Lambda`).

## Pipeline Changes
1. **Parser**: Recognize `*fn` expressions by borrowing the existing function parser with a flag for expression context. Keep placeholder nodes as-is initially.
2. **Placeholder lowering pass** (post-parse, pre-semantic):
   - Walk expressions; when encountering placeholder refs, wrap the smallest containing expression that can serve as a callable argument (e.g., the argument expression inside `foo(bar)` or RHS of binding when explicitly marked) into a synthesized lambda.
   - Replace placeholder refs with identifier expressions referencing synthesized parameters.
   - Record diagnostics if placeholders appear where no callable context exists.
3. **Semantic**:
   - Treat lambdas like nested functions: push new scope, validate parameters, etc.
   - Lambdas capture surrounding bindings by reference (initially immutably) and store capture list in a new semantic structure `LambdaClosure`.
4. **Codegen**:
   - Represent lambdas as LLVM functions taking captures as additional arguments; emit closure structs with function pointer + captured values.
   - Expose runtime trampolines `coral_call_lambda(closure, args...)` so list helpers can invoke them.

## Runtime ABI
- Define `struct CoralClosure { void* func_ptr; void* env_ptr; }`.
- Provide helpers to allocate closure envs, retain captured values, and invoke closures with N arguments (start with 1 and 2 for map/reduce).

## Diagnostics & Tests
- Parser tests for lambda literal syntax and placeholder usage.
- Semantic tests ensuring duplicate placeholder indices or unused placeholders report errors.
- Snapshot updates showing lambda nodes.
- Smoke test lowering `list.map(*fn(x) x + 1)` end-to-end.
